The Data

Example

Bad
data_final.xlsx T1 T2 T3

No one knows what this means.

Good
20231210_temperature.csv time_min, temperature_C 0, 22.1 10, 23.5

Then, add a short explanation into the README.txt :

README.txt This dataset contains temperature measurements collected in the Loire Estuary using a temperature sensor. Measurements were recorded every 10 minutes in degrees Celsius (°C).

Now anyone can use it.

If you have many files Example of how to structure the folder:

Loire_Estuary/
│
├── Field_data/
├── Code/
└── Notes/

Keep it simple, but do not mix everything in one place.

File name

A meaningful file naming is really helpful to make others follow your process and your work. Example of a good practice of naming files are:

  1. A maximum of 25 characters should be used for file names.
  2. Accents, symbols, special characters, and spaces are to be excluded.
  3. The file name is separated by underscore “_” and not space “ ”.
  4. The structure is to follow a logical order:
  • Creation date: formatted as YYYYMMDD.
  • Content description.
  • Version.
  • Name or initials of creator.
  • Project number, name, or code.
  • Avoid terms such as "draft" or "final".

Example: 20261210_DMP_v7_lp.pdf Underscores are more interoperable. They are easier for machines to read and process, while spaces often create errors.

Format

Here is a list of some open formats:

If you have… Use this (recommended) Avoid when possible
Tables / data .csv, .tsv .xlsx only
Text / notes .txt, .md, .xml .docx only
Documents .pdf, .odt locked formats
Images .png, .tif, .jpg proprietary formats
Data (structured) .json, .xml unclear formats
Geospatial .geojson, .kml, .shp software-specific only
Audio .wav, .mp3, .flac proprietary formats
Video .mp4, .webm uncommon codecs
Archives .zip, .tar.gz unclear compression

Geospatial Data

If you have… Use this (recommended) Avoid when possible
Vector data .geojson, .shp, .gpkg proprietary-only formats
Raster data .tif (GeoTIFF) unclear or compressed only
Maps (web) .geojson, .kml platform-locked formats
Coordinates data .csv (with lat/long) missing coordinate system

What good geospatial data looks like:

  • coordinates are included (lat/long or projection)
  • coordinate system is specified (e.g. WGS84 / EPSG:4326)
  • file can be opened in common tools (QGIS, Python, R)
  • variables are clearly named
  • metadata explain how data were collected

Code

Here is example of recommended formats for code.

If you have… Use this Avoid
Scripts .py, .R, .m, .sh compiled only files
Notebooks .ipynb no explanation
Documentation .md, .txt no README
Environment requirements.txt, .yml not specified
Version control Git (GitHub, GitLab) local only
Archive .zip, .tar.gz scattered files

What good code looks like:

  • the code runs
  • the steps are clear
  • there are comments
  • dependencies are listed
  • someone else can execute it

Minimal example

project/
├── data/
├── code/
│   ├── analysis.py
│   └── README.md
└── results/

Add a note inside README.txt:

Run analysis.py using Python 3.10 Install dependencies from requirements.txt
Remember: If someone cannot run your code and obtain the same result, it is not ready to share.
