Metadata

Metadata are simply the information that explains your data. They allow someone else to understand what you did, without asking you. A useful way to structure this is through Dublin Core (DCMI). You can create a .xml or .json file (editable with a text editor). Example:

Title:
Creator:
Subject:
Description:
Publisher:
Contributor:
Date:
Type:
Format:
Identifier:
Source:
Language:
Relation:
Coverage:
Rights:

What does “embedded metadata” mean? Embedded metadata are written inside the file itself. This is possible for some formats:

  • images (.tif, .jpg) = metadata stored in the file
  • geospatial data (.tif, .shp) = includes coordinate system, projection
  • PDFs = author, title, keywords
  • some scientific formats (.nc, .hdf5)

In these cases, the information travels with the file.

When metadata are separate

For many datasets (especially .csv), metadata are not embedded. In that case, you provide them as:

  • a README file
  • a metadata file (.json, .xml)
  • fields filled directly in the repository (e.g. Zenodo form)

Which one should you choose?

  • If your format supports it = embed metadata
  • If not = use README + metadata file
  • In any case, fill metadata form in the repository when you deposit your data
All you need to remember: Metadata do not have to be in one place, what matters is that they exist and are clear.
