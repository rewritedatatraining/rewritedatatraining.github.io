README

A README is the simplest and most important document in your dataset. It is where you explain your data in plain language. While metadata is machine-readable, a README is human-readable. Think of it as the instruction manual for your data. Like a letter to other people, but it contains the story of your data. Without a README, even well-structured data can become difficult to understand.

What a README does A README allows someone else to:

  • understand what the dataset is
  • know how it was created
  • interpret the variables
  • use the data without asking you

It is the bridge between your data and the person who will reuse them.

What to include A README does not need to be long. It just needs to be clear. At minimum:

  • what the dataset contains
  • how the data were collected (e.g. instrument, technique, method)
  • what each variable means
  • units and formats
  • any processing steps
  • who to contact (optional)
  • license

Example README.txt

Title: Temperature dataset, Loire Estuary, April 2024

Description:
This dataset contains temperature measurements collected in the Loire Estuary between 2024-04-10 and 2024-04-15.

Measurements were recorded every 10 minutes in degrees Celsius (°C).

Location:
Loire Estuary, France
Coordinates: 47.25 N, -2.10 W

Instrument:
Temperature sensor (HOBO data logger)

Method:
The sensor was installed at 1 meter above ground level and recorded temperature automatically every 10 minutes.

Calibration:
The sensor was calibrated before deployment according to manufacturer specifications.

Variables:
time = timestamp (YYYY-MM-DD HH:MM)
temperature_C = temperature in degrees Celsius

Format:
CSV files

License:
CC BY 4.0
